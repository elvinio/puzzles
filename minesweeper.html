Minesweeper

Goal

Reveal every safe cell on the grid without clicking a hidden bomb. You win when all non-bomb cells are uncovered.

Controls

  • 👆Tap / Click a cell to plant or remove a 🚩 flag marking a suspected bomb.
  • 👆👆Double-tap (mobile) or double-click / right-click (laptop) to dig up a cell and reveal what's underneath.
  • 🙂Press the smiley button at any time to start a fresh game.

Reading the Numbers

  • 1️⃣A number tells you exactly how many bombs are hiding in the 8 cells that touch it (sides and corners).
  • ⬜A blank cell has zero bombs nearby — the game automatically opens all its neighbours for you.
  • 💣Click a bomb and it's game over — all bombs are revealed.

Difficulty

  • 🟢Easy — 9×9 grid, 10 bombs. Good for learning.
  • 🟡Medium — 16×9 grid, 25 bombs. A real challenge.
  • 🔴Hard — 16×16 grid, 40 bombs. Expert territory.
💡 Safe first click — your very first tap is always safe. Bombs are placed after you click, so you can never lose on move one.

The Grid — a flat array

The board is stored as a plain JavaScript array. Each item in the array represents one cell and holds four pieces of information:

// One cell object { mine: false, // is there a bomb here? revealed: false, // has the player dug it up? flagged: false, // has the player flagged it? adjacent: 0 } // how many bombs touch it?

A 9×9 grid has 81 cells stored as cells[0] … cells[80]. To find the row and column from an index: row = Math.floor(index / cols), col = index % cols.

Safe First Click

Bombs aren't placed when the game starts — they're placed the moment you make your first tap. The code builds a safe zone (the clicked cell plus all 8 neighbours) and shuffles the remaining cells randomly, picking the first N as bomb locations.

// Fisher-Yates shuffle — fair random order for (let i = available.length - 1; i > 0; i--) { const j = Math.floor(Math.random() * (i + 1)); [available[i], available[j]] = [available[j], available[i]]; }

Counting Neighbours

After placing bombs, the code loops over every safe cell and checks all 8 surrounding cells. Each bomb neighbour adds 1 to that cell's adjacent count. This is where the numbers 1–8 come from.

for (let dr = -1; dr <= 1; dr++) { for (let dc = -1; dc <= 1; dc++) { // check the cell at (row+dr, col+dc) if (neighbour.mine) count++; } }

Flood Fill — opening blank areas

When you click a cell with adjacent = 0 (no bomb neighbours), there's no useful number to show. Instead of making you click every blank cell one by one, the game uses a BFS queue (Breadth-First Search) to automatically open all connected blank cells and the numbered border around them.

const queue = [startIndex]; while (queue.length) { const cur = queue.shift(); // take from front cells[cur].revealed = true; if (cells[cur].adjacent === 0) { // add all unvisited neighbours queue.push(...neighbours); } }

Double-Tap Detection

Touch screens don't have a right-click, so digging works by detecting two taps on the same cell within 300 ms. The code remembers the last tapped cell and the timestamp. A single tap waits 300 ms before flagging, so a quick double-tap can intercept it and dig instead.

if (idx === lastTapCell && timeSince < 300) { reveal(idx); // double-tap → dig } else { setTimeout(() => toggleFlag(idx), 300); // wait for possible 2nd tap }

Drawing the Board

The board is a CSS Grid where each cell is a <div>. The cell size is calculated dynamically so the board always fills the screen width. When the window is resized, renderBoard() runs again and updates --cell-size automatically.

const cellSize = Math.floor(boardWidth / cols); board.style.gridTemplateColumns = `repeat(${cols}, ${cellSize}px)`;
